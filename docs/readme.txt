1. TaskFlowZero について
================================================================================

TaskFlowZero は、単一の HTML ファイルで動作するローカル型プロジェクト管理ツールです。
インターネット接続・サーバー・インストール作業は一切不要で、
HTML ファイルをブラウザで開くだけですぐに使用できます。

データはプロジェクトごとに JSON ファイルとして保存されるため、
フォルダをコピーするだけでバックアップが完了します。

主な機能:
  - 複数プロジェクトの管理
  - タスクのリスト / カンバン / ガント / バーンダウン表示
  - マイルストーン（マイルストーン → タスクの階層構造）
  - タスクのチェックリストと進捗率
  - コメント＆アクティビティログ（変更履歴の自動記録）
  - URLでタスクを直接共有
  - 複数人利用対応（3-wayマージによる競合検知）
  - ライト / ダークテーマ切り替え
  - 自動保存
  - プラグインによる機能拡張（15章参照）

2. 動作環境
================================================================================

【必須】
  Google Chrome または Microsoft Edge（最新版推奨）

【非対応】
  Firefox / Safari / モバイルブラウザ

  理由: データファイルの読み書きに「File System Access API」を使用しており、
        この API は Chrome / Edge 以外では動作しません。

3. フォルダ構成
================================================================================

TaskFlowZero はデータを「フォルダ」単位で管理します。
プロジェクトごとに個別の JSON ファイルが生成されます。

  📁 （データフォルダ / 共有フォルダ）
  ├── project_P1.json          ← プロジェクトP1のデータ（自動生成）
  ├── project_P2.json          ← プロジェクトP2のデータ（自動生成）
  ├── taskflow_data.json       ← 旧形式ファイル（あれば自動移行）
  ├── marked.min.js            ← オフライン対応する場合のみ
  └── chart.min.js             ← オフライン対応する場合のみ

  📁 （taskflowzero.html と同じフォルダ）
  ├── taskflowzero.html        ← 本体（これをブラウザで開く）
  └── 📁 plugin/               ← プラグインフォルダ（任意）
      ├── plugins.js           ← プラグインマニフェスト（自動生成）
      ├── plugin_svn.js        ← SVN連携プラグイン（例）
      └── plugin_lang.js       ← 言語切り替えプラグイン（例）

【データフォルダと本体フォルダは別でも構いません】
  taskflowzero.html は任意の場所に置き、起動時にデータフォルダ（共有フォルダ等）
  を選択する形式です。

【旧形式（taskflow_data.json 単一ファイル）からの移行】
  旧形式のデータフォルダを開くと自動的に検出されます。
  次回保存時にプロジェクトごとの分割形式へ自動移行されます。

【オフライン（イントラネット）環境での利用】
  インターネットに接続できない環境では、以下の2ファイルを
  データフォルダに置いてください。

    marked.min.js  → https://cdn.jsdelivr.net/npm/marked/marked.min.js
    chart.min.js   → https://cdn.jsdelivr.net/npm/chart.js/dist/chart.umd.min.js
                     （ファイル名を chart.min.js に変更して保存）

  ファイルが存在しない場合は CDN からの読み込みを試みます。
  両方失敗した場合、Markdown プレビューとバーンダウンチャートが動作しません。

4. はじめかた
================================================================================

【初回起動】

  1. taskflowzero.html をダブルクリック（または Chrome/Edge にドラッグ）して開く

  2. 起動画面が表示されます。以下いずれかを選択してください:

     ● 新規作成
       「✨ 新規作成」ボタンをクリック
       データを保存するフォルダ（共有フォルダ等）を選択する
       プロジェクト管理画面が開きます

     ● 既存フォルダを開く
       「📂 別のフォルダを開く」ボタンをクリック
       既存のデータフォルダ（project_P*.json が入ったフォルダ）を選択する

【2回目以降】

  前回使用したフォルダが自動的に認識されます。
  「↩ 開く」ボタンをクリックするだけで再開できます。
  ※ ブラウザにアクセス許可を求められた場合は「許可」を選択してください。

5. データファイルについて
================================================================================

TaskFlowZero のデータはプロジェクトごとに個別の JSON ファイルとして保存されます。
ファイル名は自動で割り当てられたプロジェクト番号に基づきます（例: project_P1.json）。

【保存タイミング】
  - タスクやプロジェクトを変更すると約1.2秒後に自動保存されます
  - Ctrl + S（Mac: ⌘ + S）で手動保存も可能です
    ※ 手動保存時は他のユーザーの変更との競合チェックも行われます（6章参照）

【バックアップ】
  データフォルダごとコピーするだけでバックアップが完了します。
  例: taskflow_backup_20260314/ というフォルダ名でコピーして保管

【プロジェクト番号の自動採番】
  プロジェクトには自動で番号が割り当てられます（P1, P2, P3...）。
  旧形式の JSON を開いた際に番号が未設定の場合は自動採番されます。

6. 複数人での利用（保存競合の検知とマージ）
================================================================================

TaskFlowZero は同じデータフォルダを複数人で共有して利用できます。
Ctrl + S による手動保存時に「3-wayマージ」と呼ぶ仕組みで競合を検知します。

【基本的な使い方】
  1. 共有フォルダ（社内ネットワーク, OneDrive, Dropbox 等）を
     データフォルダとして指定する
  2. 各自の PC から同じフォルダを開く
  3. 編集後 Ctrl + S で保存すると、他者の変更が自動で取り込まれる

【保存時の動作】

  ● 他の人が別のタスク・別のプロジェクトを変更していた場合
    → 競合なし。自動マージして保存されます。
       トーストに「他の変更を自動マージ」と表示されます。

  ● 同じタスクの同じフィールドを別々に変更していた場合
    → 競合あり。競合ダイアログが表示されます。
       左列（自分の変更）と右列（相手の変更）を比較し、
       どちらを残すか行ごとにボタンで選択できます。
       「すべて自分を残す」「すべて相手を残す」の一括選択も可能です。

  ● コメント・アクティビティログ・チェックリスト
    → 常に両方を取り込んでマージされます（競合扱いにはなりません）。

【OneDrive / Dropbox の競合コピーについて】
  OneDrive や Dropbox は同時編集時に「競合コピー」ファイルを自動生成することがあります。
  （例: project_P1 (田中 の競合コピー 2026-03-14).json）
  TaskFlowZero はフォルダを開いた際にこれらを自動検出し、3-wayマージを試みます。
  競合のない変更は自動マージ、同じフィールドへの競合は手動解決ダイアログを表示します。
  解決後、競合コピーファイルは自動的に削除されます。

【他の人の変更を取り込むタイミング】
  手動保存（Ctrl+S）のほか、以下の操作でも自動的に取り込まれます:

    - ビュータブを切り替えたとき（リスト↔ガント↔カンバン 等）
    - タスク詳細パネルを開いたとき
    - ヘッダーの「🔄 更新」ボタンをクリックしたとき

  ※ タスクパネルに未保存の変更がある状態でタブを切り替えた場合は
    上書き防止のためマージをスキップします。

【注意事項】
  - 自動保存（1.2秒後）はマージを行わず高速書き込みのみ行います。
    他の人の変更を確実に取り込みたいときは Ctrl+S を使ってください。
  - 全く同時（ミリ秒単位）の書き込みは防ぎきれない場合があります。

8. プロジェクト管理
================================================================================

【作成】
  プロジェクト一覧画面の「＋ 新規プロジェクト」ボタンをクリック
  名前・説明・期間・メンバー・カラーを設定する

【プロジェクト番号】
  各プロジェクトには自動で番号が割り当てられます（P1, P2, P3...）。
  この番号はタスクの直接URLに使用されます（13章参照）。
  既存の JSON を開いた際に番号が未設定の場合は自動採番されます。

【検索・絞り込み】
  プロジェクト一覧画面では、プロジェクト名やメンバー名で検索できます。

【完了（アーカイブ）】
  プロジェクトが完了したら「📦 完了」ボタンを押します。
  アーカイブされたプロジェクトはグレーアウトされ、下部に分類されます。
  「♻️ 未完了に戻す」でいつでも復元できます。

【編集・削除】
  プロジェクトカードの「✏️」「🗑」ボタンで操作できます。
  ※ 削除は取り消せません。

9. タスク管理
================================================================================

【作成】
  画面上部「＋ タスク追加」ボタン、またはカンバンビューの「＋ タスクを追加」から

  設定できる項目（追加モーダルの並び順）:
    タイトル（必須）
    ステータス          /  担当者
    開始日              /  期限
    ストーリーポイント  /  マイルストーン
    ラベル（色付きタグ、自由に追加可）

【ステータスと進捗】
  ステータスを「完了」に変更すると完了日が自動セットされます。
  チェックリストの完了率はステータス「完了」にした時点で100%になります。

================================================================================
10. マイルストーン
================================================================================

マイルストーンはタスクをグループ化する単位です。
プロジェクト → マイルストーン → タスクの3階層で管理します。

【作成・編集】
  プロジェクト画面上部の「🏁 マイルストーン」ボタンをクリック

  設定できる項目:
    名前 / 説明 / 開始日 / 終了日 / カラー

【並び替え】
  マイルストーン一覧でドラッグ&ドロップによる並び替えが可能です。
  並び順はリストビューとガントチャートに反映されます。

================================================================================
11. ビュー一覧
================================================================================

プロジェクト画面上部のタブでビューを切り替えます。
タブを切り替えるたびに他のユーザーの変更が自動で取り込まれます。

────────────────────────────────────────
11-1. リストビュー
────────────────────────────────────────

タスクをマイルストーン別の表形式で表示します。

【タブ】
  未完了 / 完了 / 全て の3つのタブで表示を切り替えられます。
  タブ行はスクロールしても画面上部に固定されます。

【フィルター】
  - テキスト検索（タイトル部分一致）
  - マイルストーン絞り込み
  - 担当者絞り込み
  - ラベル絞り込み
  - 期限絞り込み（開始前 / 期間内 / 期限切れ / 期限なし）

【グループの折りたたみ】
  マイルストーン行をクリックすると、そのグループのタスクを折りたたみます。

【マイルストーン行の色分け（ライトモード）】
  マイルストーン行は青色背景で表示され、タスク行と明確に区別できます。

【マイルストーンのツールチップ】
  説明が設定されているマイルストーン行には 💬 アイコンが表示されます。
  行にマウスを乗せると 0.2秒後 に説明内容がツールチップで表示されます。

────────────────────────────────────────
11-2. カンバンボード
────────────────────────────────────────

タスクをステータス別（To Do / 進行中 / レビュー / 完了）のカードで表示します。

【操作】
  - カードをドラッグ&ドロップでステータス変更
  - カードをクリックで詳細パネルを開く

【スクロール】
  タスクが多い列はカード一覧エリアのみスクロールされます。
  「＋ タスクを追加」ボタンは各列の下部に常時表示されます。

────────────────────────────────────────
11-3. バーンダウンチャート
────────────────────────────────────────

スプリント期間中のストーリーポイント消化状況をグラフで確認できます。

  - 灰色の線: 理想消化ライン
  - 青色の線: 実際の残ポイント推移

タスクに「ストーリーポイント」が設定されていると、
より正確な進捗をグラフで確認できます。

────────────────────────────────────────
11-4. ガントチャート
────────────────────────────────────────

マイルストーンとタスクを時系列バーで可視化します。

【表示範囲の切り替え】
  1ヶ月 / 3ヶ月 / 6ヶ月 / 1年から選択できます。

【起点の設定】
  「今日起点」または「プロジェクト開始日起点」を選択できます。

【バーのドラッグ操作】
  左端をドラッグ  → 開始日を変更
  右端をドラッグ  → 終了日（期限）を変更
  中央をドラッグ  → 期間全体をずらす
  ドラッグ中は日付のツールチップが表示されます。

【進捗率の計算（チェックリスト連動）】
  ステータス「完了」           → 100%
  チェックリストあり（未完了） → チェック数 ÷ 総数（最大 99%）
  チェックリストなし（未完了） → 0%
  マイルストーンバーの進捗はタスクのSP加重平均で算出されます。

【イナズマ線】
  各マイルストーンの進捗を繋いだ折れ線を表示します。
  100%完了かつ終了日が過去、または 0%かつ開始日が未来の場合は
  ドット位置を「今日」に補正します。

────────────────────────────────────────
11-5. Wiki
────────────────────────────────────────

プロジェクトごとの自由記述スペースです。Markdown 形式で記述できます。
右パネルに自動生成された目次（TOC）が表示されます。
本文をスクロールすると、現在表示中の見出しが目次でハイライトされます（スクロールスパイ）。
目次の項目をクリックすると、対応する見出しへスムーズにスクロールします。

================================================================================
12. タスク詳細パネル
================================================================================

タスクをクリックすると画面右側に詳細パネルが開きます。
パネルを開いた際、他のユーザーの最新変更が自動で取り込まれます。

【パネル幅の変更】
  パネル左端をドラッグすると幅を変更できます（最小幅: 640px）。
  設定した幅はブラウザに記憶されます。

【チェックリスト】
  - Enter で項目追加
  - テキスト部分をクリックしてインライン編集（Enter/blur で保存、Esc でキャンセル）
  - ⠿ ハンドルをドラッグして並び替え
  - 進捗率 = チェック数 ÷ 総数（最大 99%、完了ステータスで 100%）

【詳細（Markdown）】
  「プレビュー」タブがデフォルトで表示されます。
  「編集」タブで入力、「プレビュー」タブで確認。
  プレビューエリアの高さはドラッグで変更可能で、ブラウザに記憶されます。

【コメント＆アクティビティ フィルター】
  すべて / 💬 コメントのみ / 🔄 アクティビティのみ の3種類。
  設定はブラウザに保存されます。

  コメントは Markdown 形式で記述でき、プレビュー表示にも対応しています。

  自動記録される操作:
    ステータス / 担当者 / 期限の変更、詳細内容の更新、チェックリストの操作

13. URLでタスクを直接開く
================================================================================

【URL形式】
  file:///C:/Users/.../taskflowzero.html#P1-42
  P1 → プロジェクト番号 / 42 → タスク番号

タスク詳細パネルの「🔗 URLコピー」ボタンでクリップボードにコピーできます。
URLはローカルファイルパスを含むため、別PCへそのまま送っても使用できません。
同じ共有フォルダに taskflowzero.html を置き、同じパスでアクセスしている場合は動作します。

15. プラグイン
================================================================================

TaskFlowZero はプラグインで機能を拡張できます。
プラグインの管理はヘッダーの「🔌」ボタンから開くプラグイン管理モーダルで行います。

【プラグインのインストール】
  1. ヘッダーの「🔌」ボタンをクリック
  2. 初回はプラグインフォルダ（plugin/）を選択する
  3. .js ファイルをドラッグ&ドロップ、またはファイル選択でインストール
  4. 「保存」ボタンで plugins.js に反映し、ページをリロードすると有効になります

【plugins.js について】
  プラグインフォルダ内の plugins.js がプラグインのマニフェストファイルです。
  モーダルから「保存」すると自動的に書き換えられます。
  手動で編集することも可能です。

【プラグインの並び替え・削除】
  モーダル内のリストで ⠿ ハンドルをドラッグして読み込み順を変更できます。
  🗑 ボタンで一覧から削除できます（ファイル自体は残ります）。

現在提供されているプラグイン:

  plugin_lang.js   言語切り替え（日本語 ⇔ 英語）
                   → 詳細は plugin/README_plugin_lang.txt 参照

  plugin_svn.js    SVN連携（コミットとタスクの紐付け）
                   → 詳細は plugin/README_plugin_svn.txt 参照

================================================================================
16. ショートカット・操作ヒント
================================================================================

  Ctrl + S（Mac: ⌘ + S）  手動保存（競合チェックも実行）
  Escape                   開いているパネル・モーダルを閉じる

  【ヘッダーボタン】
  - 🔄 更新: 他のメンバーの変更を即時取得（Ctrl+S を押さずに確認したいとき）
  - 📂 フォルダ変更: 別のデータフォルダに切り替える
  - 🔌 プラグイン: プラグイン管理モーダルを開く

  【その他のヒント】
  - タスクの「#番号」は自動採番されます（変更不可）
  - ラベルを入力欄に打ち込んで Enter を押すだけで追加できます
  - マイルストーン一覧でドラッグ&ドロップによる並び替えが可能です
  - ガントのバーをドラッグすると日付を直接変更できます
  - チェックリストの ⠿ ハンドルをドラッグすると並び替えられます
  - コメントフィルター / プレビュー高さの設定はブラウザに保存されます

17. よくある質問
================================================================================

Q. フォルダを開こうとすると「アクセスが拒否されました」と表示される
A. Chrome のセキュリティ制約により、前回のフォルダアクセス許可が
   セッション間で失われることがあります。
   「📂 別のフォルダを開く」から再選択してください。

Q. 複数人で同時に保存するとどうなりますか？
A. Ctrl+S を押したタイミングでファイルを再読み込みし、競合を検知します。
   別タスク・別プロジェクトへの変更は自動マージされます。
   同じフィールドを変更していた場合のみダイアログが表示されます。

Q. OneDrive / Dropbox の競合コピーファイルが発生した場合は？
A. 次回フォルダを開いたときに自動検出し、3-wayマージを実行します。
   競合のない変更は自動マージ、競合がある場合は手動解決ダイアログが表示されます。
   解決後、競合コピーファイルは自動的に削除されます。

Q. スマートフォンやタブレットで使えますか？
A. 対応していません。PC の Chrome / Edge 専用ツールです。

Q. チャートにデータが表示されない
A. バーンダウンチャートはストーリーポイントが設定されたタスクが必要です。
   ガントチャートはタスク・マイルストーンに開始日と期限が必要です。

Q. Markdown プレビューやバーンダウンチャートが表示されない
A. インターネット未接続環境では marked.min.js / chart.min.js が必要です。
   3章「フォルダ構成」の手順でファイルを配置してください。
